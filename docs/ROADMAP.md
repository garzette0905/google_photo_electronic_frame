# 제품 로드맵 / 기능 기획 노트

상용 전자액자(Aura · Nixplay · Skylight · Pix-Star, 그리고 구글 Nest Hub · 삼성 The Frame)와
비교한 기능 격차 분석과 향후 개발 우선순위 정리. 핵심은 "어떤 기능이 없느냐"보다
**"이 제품의 정체성을 결정하는 한 가지 결정"**이다.

---

## 가장 중요한 결정: 원격 사진 공급 (제품의 성패를 가름)

상용 전자액자가 파는 것은 "사진 보는 기기"가 아니라 **"떨어져 사는 가족이 부모님 액자에
사진을 원격으로 계속 보내주는 경험"**이다. Aura · Skylight의 리뷰가 극찬받는 이유의 90%가 이것.

현재 제품은 여기서 두 가지가 막혀 있다:

- **자동 동기화 없음**: 새 사진을 넣으려면 수동으로 "다시 선택" (Picker API 구조적 한계)
- **가족이 각자 보낼 수 없음**: 로그인한 한 계정의 사진만 표시

> ✅ **1차 진전 (PWA 공유 타깃 구현 완료)** — 앱을 PWA로 만들어 안드로이드에서
> 구글 포토 등의 "공유" 대상 앱으로 등록되게 했다. 가족이 폰에서 사진/동영상을
> **공유 → 기억 조각 이어주기**로 보내면 로그인 없이 바로 재생된다. 단, 안드로이드
> Chrome 전용·PWA 설치 필요·HTTPS 필요이며, 자동(무설정) 동기화는 아니라 아래
> 방향(C/B)의 대체가 아닌 보완 진입로다.

이 둘은 뿌리가 같고, **아키텍처 선택**이 필요하다:

| 방향 | 방법 | 장단점 |
|---|---|---|
| **A. 구글 Ambient API 파트너 승인** | 인물/앨범 자동 동기화 | 진짜 자동, but 파트너 심사 통과 어려움 |
| **B. 자체 업로드 백엔드 구축** | 가족이 내 사이트에 사진 업로드 → 액자가 표시 | 완전한 제어(공유·캡션·다중액자), but 저장소 비용·개발량 큼 |
| **C. 구글 공유앨범 활용** | 가족이 공유앨범에 추가 → 액자가 그 앨범 표시 | 지금 구조에 가장 가깝고 저비용, but 사용자가 공유앨범 세팅 필요 |

**추천: C → B 순의 단계적 접근.** 우선 C(공유앨범 연동 + 주기적 자동 새로고침)로
"자동 동기화 + 가족 공급"을 저비용으로 구현해 시장 반응을 보고, 반응이 좋으면
B(자체 백엔드)로 넘어가는 것이 리스크가 낮다.

## Quick Wins — 적은 노력, 큰 체감 (우선 구현 추천)

상용 액자 대비 "싸구려 티" 나는 부분을 없애는 것들:

1. **세로 사진 처리** — 지금은 세로 사진 양옆이 검은 여백. 상용은 **흐린 배경(같은 사진 blur)** 을
   깔거나 **세로 2장을 나란히** 보여준다. 이거 하나만 해도 완성도가 확 올라감. (몇 시간 작업)
2. **화면 켜짐/꺼짐 스케줄** — "밤 11시~아침 7시 화면 끔." 액자의 필수 기능인데 지금 없음.
   웹에서 화면을 검게 + 밝기 0으로 처리하면 됨.
3. ✅ **전환 간격 · 전환 효과 설정 UI** _(구현 완료)_ — 정보 패널에서 전환 간격(3~30초)
   리스트박스 선택 + 페이드/슬라이드/Ken Burns 효과 라디오 선택. 설정은 localStorage에 저장.
4. ✅ **Ken Burns 효과** _(구현 완료)_ — 효과 선택의 한 옵션으로 통합. 표시되는 동안
   사진이 천천히 줌·팬하며, 애니메이션 길이는 전환 간격과 자동 동기화.
5. ✅ **시계 · 날씨 위젯 오버레이** _(구현 완료)_ — 우측 상단. 시계는 항상 표시,
   날씨는 위치 권한 + Open-Meteo(무료·키 불필요, 좌표는 소수 2자리로만 전송)로 best-effort.
   전체화면에서도 표시. "시계·날씨 표시" 토글로 on/off.
6. ✅ **슬라이드쇼 제목 오버레이** _(구현 완료)_ — 좌측 상단에 사용자 지정 제목 표시.
   전체화면에서도 유지되며 localStorage에 저장. (로드맵 항목 외 추가 구현)

## 중간 난이도 — 감성/참여 기능

7. **"몇 년 전 오늘"** — 오늘 날짜의 과거 사진 우선 노출. 이미 촬영일 데이터가 있어 구현 쉽고 감동 큼.
8. **캡션/사진 설명** — 사진에 "2015년 제주도" 같은 메모.
9. **동영상 · 움직이는 사진 재생** — ✅ **동영상 재생 구현 완료**(음소거 자동재생, 종료 시
   자동 전환, `/video` 프록시로 Range 지원). 움직이는 사진(Motion Photo) 재생은 아직 미구현.
10. **다중 액자 관리** — 한 계정으로 여러 액자(거실 · 부모님댁) 다른 사진 세트 운영.

## 웹앱으로는 어려운 것 (쫓지 말 것)

상용 하드웨어 액자의 아래 기능들은 순수 웹앱으로 불가하거나 제한적. 이걸 못 한다고 약점이
아니라, 오히려 **"하드웨어 없이 시작"이 이 제품의 차별점**이다:

- 모션 센서 (사람 없으면 화면 끔)
- 주변광 밝기 자동조절
- 물리 터치 제스처 (태블릿이면 일부 가능)

## 상품기획 관점 종합 제언

**포지셔닝**: "하드웨어 안 사고, 집에 남는 태블릿/구형 PC/TV로 무료로 시작하는 가족 사진 액자."
Aura(기기값 20만원+)의 진입장벽을 없앤 것이 무기.

**로드맵 추천 순서:**

1. **Quick Wins** → 완성도를 상용 수준으로 끌어올려 초기 사용자 이탈 방지
   - ✅ 완료: 전환 간격·효과 설정, Ken Burns, 시계·날씨, 제목 오버레이, 동영상 재생
   - ⬜ 남음: **세로 사진 흐린 배경 처리**, **취침 스케줄(화면 켜짐/꺼짐)** — 다음 우선순위
2. **"몇 년 전 오늘"** → 저비용 감성 훅으로 입소문 유도
3. **자동 동기화(방향 C)** → 재방문·리텐션의 핵심. 이게 되면 "한 번 설정하면 알아서 되는 액자" 완성
4. 반응 확인 후 **자체 백엔드(방향 B)** 로 가족 공유·원격 전송·캡션까지 확장 = **유료화 지점**

**우선순위 한 줄 요약:**
세로 사진 처리 + 취침 스케줄(당장 체감) → 몇 년 전 오늘(감성) → 자동 동기화(리텐션) → 가족 원격 공유(사업화)

> 첫 타자 추천: **세로 사진 흐린 배경 처리 + 취침 스케줄** — 노력 대비 체감이 가장 큼.
